# Metadata

Most updatable Balancelynx objects--such as Account, AccountGroup, and Transaction--have a metadata parameter.
You can use this parameter to attach arbitrary key-value data to these Balancelynx objects.

Metadata is useful for storing additional, structured information on an object. For example, you could store your user's corresponding unique identifier from your system on a Balancelynx `Account` object.

<Callout type="info">
    Balancelynx uses the `name` key to display Accounts and Commodities in the admin dashboard.
</Callout>
